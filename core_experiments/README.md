# Core Experiments: Offline Benchmark Pipeline

## Module Overview and Experimental Scope

This module implements the offline benchmark pipeline of Ca-Bench. It covers graph construction from captured traffic, strict protocol split preparation, centralized model training, baseline comparison, federated evaluation, and generation of paper-ready tables and figures.

The pipeline consumes the packet captures generated by the Mininet module and produces the quantitative artifacts reported in the paper.

## Environment and Dependency Preparation

Execution is intended for Ubuntu/Linux with a Python environment containing the project dependencies. The scripts default to the interpreter path `/home/user/miniconda3/envs/DL/bin/python`; users may override this path through the `PY_BIN` environment variable.

The offline pipeline depends on the following Python packages:

- `numpy`
- `matplotlib`
- `torch`
- `torch-geometric`
- `scikit-learn`
- `flwr`
- `scapy`

There is currently no repository-provided `requirements.txt` or `environment.yml`; the environment must therefore be prepared manually.

Before running this module, ensure that the scenario captures already exist under:

```bash
../mininet_testbed/real_collection/scenario_*/full_arena_v2.pcap
```

These packet captures are not shipped with the repository and must be generated locally through the Mininet module first.

## Execution Entry

Run the complete offline pipeline:

```bash
PY_BIN=/path/to/python bash run_full_eval.sh
```

Run the two major suites separately if partial reproduction is preferred:

```bash
PY_BIN=/path/to/python python main_suite.py --project-dir "$(pwd)" --real-collection-dir ../mininet_testbed/real_collection --output-root ../paper_artifacts/runs
```

```bash
PY_BIN=/path/to/python python system_suite.py --project-dir "$(pwd)" --real-collection-dir ../mininet_testbed/real_collection --output-root ../paper_artifacts/runs
```
